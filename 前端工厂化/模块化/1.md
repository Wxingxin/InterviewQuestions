# 🎯 JavaScript ES Module 导入 / 导出 使用大全（含组合用法）

---

## 🧩 一、基本语法概览

| 分类                   | 导出语法                           | 导入语法                                 |
| -------------------- | ------------------------------ | ------------------------------------ |
| 命名导出（Named Export）   | `export const a = 1;`          | `import { a } from './m.js';`        |
| 默认导出（Default Export） | `export default function() {}` | `import fn from './m.js';`           |
| 重命名导出                | `export { a as b };`           | `import { b as myB } from './m.js';` |
| 全部导出                 | `export * from './other.js';`  | `import * as mod from './m.js';`     |
| 动态导入                 | ——                             | `import('./m.js').then(...)`         |

---

## 🧩 二、命名导出（Named Export）

可以同时导出多个变量、函数、类。

### ✅ 示例：

```js
// math.js
export const PI = 3.14;
export function add(a, b) { return a + b; }
export class Circle {}
```

### ✅ 导入方式：

```js
// main.js
import { PI, add, Circle } from './math.js';
console.log(add(2, 3)); // 5
```

---

## 🧩 三、一次性命名导出

```js
// math.js
const PI = 3.14;
const add = (a, b) => a + b;

export { PI, add };
```

> ✅ 常用于集中导出多个内容。

---

## 🧩 四、重命名导出与导入

### ✅ 导出重命名：

```js
// math.js
function sum(a, b) { return a + b; }
export { sum as add };
```

### ✅ 导入重命名：

```js
// main.js
import { add as plus } from './math.js';
console.log(plus(1, 2)); // 3
```

---

## 🧩 五、默认导出（Default Export）

### ✅ 导出单个主对象

```js
// greet.js
export default function greet() {
  console.log("Hello World!");
}
```

### ✅ 导入：

```js
// main.js
import greet from './greet.js';
greet(); // Hello World!
```

### ✅ 默认导出也可以是对象、类：

```js
// defaultExport.js
export default {
  name: "Tom",
  age: 18
};
```

---

## 🧩 六、命名导出 + 默认导出 共存

> 一个模块中 **只能有一个默认导出**，但可以同时有多个命名导出。

### ✅ 示例：

```js
// user.js
export const name = "Alice";
export const age = 20;
export default function sayHi() {
  console.log("Hi!");
}
```

### ✅ 导入方式：

```js
import sayHi, { name, age } from './user.js';
sayHi();
console.log(name, age);
```

---

## 🧩 七、导入所有成员（namespace import）

当你要导入模块中所有导出的内容，可以使用 `* as`：

```js
// math.js
export const PI = 3.14;
export function add(a, b) { return a + b; }
```

```js
// main.js
import * as math from './math.js';
console.log(math.add(1, 2)); // 3
console.log(math.PI); // 3.14
```

> 这种方式常用于工具库或命名空间管理。

---

## 🧩 八、重新导出（Re-export）

一个模块可以重新导出另一个模块的内容。

### ✅ 示例：

```js
// utils/math.js
export const add = (a, b) => a + b;
```

```js
// utils/index.js
export * from './math.js';
```

```js
// main.js
import { add } from './utils/index.js';
console.log(add(1, 2));
```

---

## 🧩 九、选择性重新导出

```js
export { add } from './math.js';
```

或重命名导出：

```js
export { add as sum } from './math.js';
```

---

## 🧩 十、导出默认并重命名（少见但重要）

```js
export { default as MyComponent } from './MyComponent.js';
```

👉 常用于 React、Vue 工程导出组件库的 index.js。

---

## 🧩 十一、动态导入（异步加载模块）

动态导入 `import()` 是一个 **函数式语法**，返回 Promise。

```js
// main.js
import('./math.js').then((mod) => {
  console.log(mod.add(1, 2));
});
```

### ✅ 配合 async/await

```js
async function load() {
  const { add } = await import('./math.js');
  console.log(add(3, 4));
}
load();
```

> ✅ 动态导入常用于懒加载、代码分割（如 React 路由懒加载）。

## 🧩 十三、导入导出混合写法总结

| 场景     | 写法                                     |
| ------ | -------------------------------------- |
| 导出多个命名 | `export { a, b, c };`                  |
| 导出单个默认 | `export default xxx;`                  |
| 同时导出两种 | `export default xxx; export { a, b };` |
| 导入默认   | `import xxx from './m.js';`            |
| 导入命名   | `import { a, b } from './m.js';`       |
| 导入并改名  | `import { a as A } from './m.js';`     |
| 导入全部   | `import * as mod from './m.js';`       |
| 动态导入   | `const mod = await import('./m.js');`  |

## 🧩 十四、ESM 导出对象的实时绑定（重要！）

```js
// counter.js
export let count = 0;
export function inc() { count++; }
```

```js
// main.js
import { count, inc } from './counter.js';
console.log(count); // 0
inc();
console.log(count); // 1 ✅ 实时绑定
```

> ESM 的变量是 “活的引用”，与 CommonJS 不同。

## 🧩 十五、常见错误与陷阱

| 错误                                             | 原因                                            |
| ---------------------------------------------- | --------------------------------------------- |
| `Cannot use import statement outside a module` | Node 环境未启用 ESM（需 `"type": "module"` 或 `.mjs`） |
| `SyntaxError: Unexpected token 'export'`       | 旧版 Node / 浏览器不支持                              |
| `default is not exported`                      | 忘记写 `export default`                          |
| `Cannot redeclare exported variable`           | 同名导出冲突                                        |

---

## 🧩 十六、在 Node.js 中使用 ESM

两种方式启用 ESM：

1. 在 `package.json` 中声明：

   ```json
   { "type": "module" }
   ```
2. 或者使用 `.mjs` 文件扩展名。
